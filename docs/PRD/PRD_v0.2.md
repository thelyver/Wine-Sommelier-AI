## 1. 제품 비전

### 브랜드
**joomoon.ai** *(joomoon = 주문, 注文)*
- 도메인 후보: `joomoon.ai` (검토 중, 변경 가능)
- 한 줄 정의: *"AI가 연결하는 B2B 주류 거래 플랫폼"*

### 배경
- 와인·수제맥주·전통주·전통증류주(이하 "주류") 유통은 여전히 전화·팩스·영업 방문 중심
- 소규모 수입상·도매상은 디지털 전환이 느리고 바이어 발굴이 어려움
- 소매상·호레카는 상품 정보 탐색과 소믈리에 전문 지식 접근이 어려움
- AI를 통해 이 격차를 좁히는 플랫폼이 필요

### 핵심 가치
1. **발견(Discovery)**: AI 소믈리에가 적합한 상품을 능동적으로 제안
2. **연결(Connection)**: 공급자와 수요자를 직접 연결하는 마켓플레이스
3. **신뢰(Trust)**: 정확한 상품 데이터와 투명한 거래 구조

## 2. 타겟 사용자

### 2-1. 수요자 (Buyer)
**누구**: 와인·전통주 소매상, 호텔·레스토랑·카페(호레카)

**핵심 니즈**
- 상황(시즌, 행사, 손님 유형)에 맞는 상품을 빠르게 찾고 싶다
- 소비자에게 상품을 설명할 수 있는 스토리와 정보가 필요하다
- 소량/다품종 주문이 가능하고 납기가 예측 가능해야 한다
- 새로운 상품 트렌드를 편하게 탐색하고 싶다

**페인포인트**
- 기존 거래처 외 신규 상품 발굴이 어려움
- 영업사원 연락을 기다려야 함
- 상품 정보가 부족하거나 산재되어 있음

---

### 2-2. 공급자 (Supplier)
**누구**: 와인 도매상, 유통업체, 수입상 (중소규모)

**핵심 니즈**
- 신규 바이어를 디지털로 발굴하고 싶다
- 상품 정보를 직접 등록·관리하고 싶다
- 재고 상황에 따라 추천 노출을 조절하고 싶다
- 주문 접수와 처리를 한 곳에서 관리하고 싶다

---

### 2-3. 내부 관리자 (Admin)
**누구**: 플랫폼 운영팀, 마케팅팀

**핵심 니즈**
- 전체 상품·사용자·주문 현황 파악
- 상품 데이터 품질 관리
- AI 소믈리에 성능 모니터링 및 프롬프트 개선
- 마케팅 큐레이션

## 3. 확정된 결정사항 ✅

### 3-1. 수익 모델 — **혼합형 (수수료 + 구독)**
- 거래액 기준 **% 수수료** (공급자 부담, 비율 추후 확정)
- 공급자 **월 구독료** (프리미엄 기능 이용 시)
- 초기 베타 기간은 무료 또는 할인 운영 가능

### 3-2. 주문/결제 MVP — **주문 요청 방식**
- 바이어가 "주문 요청" 버튼 → 공급자에게 알림
- 결제는 기존 방식(계좌이체, 세금계산서)으로 오프라인 처리
- 2차: PG 연동 (토스페이먼츠 검토)

### 3-3. 공급자 온보딩 — **직접 가입 + 초기 운영팀 병행**
- 공급자가 직접 가입 신청 → 관리자 승인
- 초기에는 운영팀이 대신 등록 병행
- 공급자가 직접 상품 관리하는 셀프서비스로 점진적 전환

### 3-4. 데이터 전략 — **가짜 데이터 확대 → 검증 → 실데이터 전환**
- 현재 231개 데이터로 계속 개발 진행
- **가짜 데이터를 추가**하여 AI 소믈리에 성능 시험
- 실제 공급자 확보 후 진짜 데이터로 단계적 교체

### 3-5. joomoonAI 앱 착수 — **웹 Phase 1 완료 후**
- 웹 플랫폼 안정화(주문 시스템 + 공급자 포털) 이후 앱 착수
- 웹 개발 과정에서 앱 기능 설계 병행 (별도 PRD 작성)

### 3-6. AI 소믈리에 우선순위 — **최우선 (서비스 핵심 차별점)**
- 주문 시스템과 AI 고도화를 동시에 진행하되
- AI 소믈리에를 **joomoon.ai의 핵심 차별점**으로 집중 투자
- 스토리텔링 · 구매 욕구 유발 · 영업 멘트 생성이 목표

### 3-7. UI/UX 개선 — **기능 우선, 디자인은 별도 크루**
- 현재는 기능 개발 우선
- 디자인 개편은 Claude Design 또는 별도 크루가 담당 예정
- 개발팀과 디자인팀 협업 시점은 Phase 2에서 조율

## 6. AI 소믈리에 고도화 방향 (상세)

> 결정사항 #6 — 최우선 차별점으로 집중 투자

### 현재 문제
- 기능적 추천에 그침 ("이 조건에 맞는 와인입니다")
- 데이터 부족으로 추천 다양성 제한
- 구매 욕구를 유발하는 감성적 표현 부재

### 목표 상태
> *"데이트 분위기에 딱 맞는 와인을 찾고 계시군요.*
> *이 부르고뉴 피노 누아는 촛불 아래서 천천히 열리는 향처럼,*
> *대화가 깊어질수록 진가를 발휘합니다."*

### 개선 방향

**데이터 품질**
- 테이스팅 노트 풍부화 (감성 언어, 비유 표현)
- 와인 스토리 추가 (생산자, 지역 특성, 빈티지 이야기)
- 음식·상황 페어링 구체화

**프롬프트 고도화**
- 상황 감지: 날씨, 계절, 인원, 분위기 등 맥락 파악
- 표현 스타일: 문학적 · 감성적 · 전문적 어조 선택
- 수요자 관점: "손님에게 이렇게 설명하세요" 영업 멘트 생성
- 구매 유도: 추천 → 재고 확인 → 주문으로 자연스럽게 연결

**성능 측정**
- 가짜 데이터 확대로 다양한 케이스 시험
- 추천 클릭률, 주문 전환율 추적
- 프롬프트 A/B 테스트

## 7. 데이터 전략 (상세)

### 단계별 계획

```
현재         → Phase 1          → Phase 2          → Phase 3
───────────────────────────────────────────────────────────
가짜 231개      가짜 데이터 확대     진짜 데이터 혼합     진짜 데이터 중심
               (500개 목표)        (공급자 입점)        (가짜 단계적 제거)
               AI 성능 시험
```

### 가짜 데이터 확대 목적
- AI 소믈리에가 다양한 상황/조건에 응답할 수 있는지 시험
- 공급자 포털 UI/UX 개발 시 실제처럼 작동하는 환경 제공
- 진짜 데이터 전환 전 품질 기준 수립

### 실데이터 전환 기준
- 공급자 최소 3곳 이상 입점 시 전환 시작
- 가짜 데이터는 `is_sample = true` 플래그로 구분하여 관리
- 운영 전환 시 sample 데이터는 비공개 처리 (DB 보존)

## 8. joomoonAI 앱 방향 (메모)
