## 1. 제품 비전

### 한 줄 정의
> "주류 공급자와 수요자를 연결하는 AI 네이티브 B2B 주류 거래 플랫폼"

### 배경
- 와인·수제맥주·전통주·전통증류주(이하 "주류") 유통은 여전히 전화·팩스·영업 방문 중심
- 소규모 수입상·도매상은 디지털 전환이 느리고 바이어 발굴이 어려움
- 소매상·호레카는 상품 정보 탐색과 소믈리에 전문 지식에 접근하기 어려움
- AI를 통해 이 격차를 좁히는 플랫폼이 필요

### 핵심 가치
1. **발견(Discovery)**: AI 소믈리에가 적합한 상품을 능동적으로 제안
2. **연결(Connection)**: 공급자와 수요자를 직접 연결하는 마켓플레이스
3. **신뢰(Trust)**: 정확한 상품 데이터와 투명한 거래 구조

## 2. 타겟 사용자

### 2-1. 수요자 (Buyer)
**누구**: 와인·전통주 소매상, 호텔·레스토랑·카페(호레카)

**핵심 니즈**
- 상황(시즌, 행사, 손님 유형)에 맞는 상품을 빠르게 찾고 싶다
- 소비자에게 상품을 설명할 수 있는 스토리와 정보가 필요하다
- 소량/다품종 주문이 가능하고 납기가 예측 가능해야 한다
- 새로운 상품 트렌드를 편하게 탐색하고 싶다

**페인포인트**
- 기존 거래처 외 신규 상품 발굴이 어려움
- 영업사원 연락을 기다려야 함
- 상품 정보가 부족하거나 산재되어 있음

---

### 2-2. 공급자 (Supplier)
**누구**: 와인 도매상, 유통업체, 수입상 (중소규모)

**핵심 니즈**
- 신규 바이어(소매상·호레카)를 디지털로 발굴하고 싶다
- 상품 정보를 직접 등록·관리하고 싶다
- 재고 상황에 따라 추천 노출을 조절하고 싶다
- 주문 접수와 처리를 한 곳에서 관리하고 싶다

**페인포인트**
- 영업 인력 의존도가 높고 비용이 큼
- 상품 소개 자료 제작이 번거롭고 업데이트가 어려움
- 바이어별 주문 이력 관리가 산만함

---

### 2-3. 내부 관리자 (Admin / Operator)
**누구**: 플랫폼 운영팀, 마케팅팀

**핵심 니즈**
- 전체 상품·사용자·주문 현황을 한눈에 파악
- 상품 데이터 품질 관리 (가짜 데이터 교체, 이미지 추가 등)
- AI 소믈리에 성능 모니터링 및 프롬프트 개선
- 마케팅 큐레이션 (특집 기획, 배너, 추천 리스트)

## 4. 핵심 기능 정의

### Phase 1 — 기반 완성 (현재 → 거래 가능한 플랫폼)

#### 4-1. 판매/주문 시스템 ⭐ 최우선
현재 결제/주문 로직이 전혀 없음. 플랫폼 비즈니스의 핵심.

**기능 범위**
- 장바구니 (수량 조절, 상품 추가/삭제)
- 주문서 작성 (배송지, 요청사항)
- 결제 연동 (1차: 세금계산서 요청 방식, 2차: PG 연동)
- 주문 상태 관리 (접수 → 처리중 → 배송중 → 완료)
- 주문 내역 조회

> **1차 MVP**: 온라인 결제 없이 "주문 요청" → 공급자 확인 → 오프라인 결제 흐름으로 시작
> **2차**: PG사 연동 (토스페이먼츠 or 포트원 권장)

#### 4-2. 공급자 포털
공급자가 직접 상품을 등록하고 주문을 받는 기능.

**기능 범위**
- 공급자 회원가입 / 승인 프로세스 (관리자 승인 방식)
- 상품 등록 폼 (기본 정보, 이미지, 재고, 가격)
- 재고 상태 관리: `판매중` / `품절` / `단종`
  - 품절·단종 상품은 AI 추천에서 제외되지만 DB에는 보존
- 주문 수신 알림 및 처리

#### 4-3. 와인 이미지
- 상품당 이미지 1장 이상 등록 (공급자 업로드)
- 미등록 시: 타입별 기본 이미지 표시
- 이미지 저장소: Cloudflare R2 또는 Railway Volume

#### 4-4. 재고 연동 추천 로직
```
현재: 모든 와인 추천 가능
개선: stock > 0 AND status = '판매중' 인 와인만 AI 추천
      → 품절/단종은 DB 보존, 추천 제외
```

### Phase 2 — 경험 고도화

#### 4-5. AI 소믈리에 고도화 ⭐ 차별점
현재 AI는 기능적 추천에 그침. 목표는 **구매 욕구를 유발하는 스토리텔러**.

**방향성**
- 단순 스펙 나열 → 감성적 언어로 와인 경험 묘사
- 상황 맥락 이해: "오늘 비 오는 날 혼자 마실 와인" 같은 감성 쿼리 처리
- 소비자에게 재판매하는 수요자를 위한 **영업 멘트 자동 생성** 기능
- 추천 → 주문으로 이어지는 **구매 행동 유도 플로우**

**기술적 개선 방향**
- 더 풍부한 상품 데이터 입력 (테이스팅 노트, 스토리, 수상 이력)
- 프롬프트 엔지니어링 고도화
- 사용자 대화 패턴 분석을 통한 지속 개선

#### 4-6. UI/UX 크리에이티브 개선
현재 디자인은 기능 중심의 평범한 구성.

**방향성**
- 와인의 감성(색감, 빈티지, 테루아)을 시각적으로 표현
- 수요자가 "발견의 기쁨"을 느낄 수 있는 인터랙션
- 공급자 포털은 효율성 중심의 클린 대시보드
- 모바일 우선 설계 (현재는 데스크탑 중심)

**구체적 개선 포인트**
- 와인 카드: 이미지 중심 레이아웃으로 전환
- 컬러 시스템: 와인 타입별 색상 언어 강화
- AI 소믈리에: 채팅 UI를 더 몰입감 있게 (타이핑 애니메이션, 와인 카드 인라인 표시)
- 홈화면: 큐레이션 스토리텔링 섹션 추가

#### 4-7. 즐겨찾기 / 자주 구매
- 수요자가 자주 주문하는 상품 빠른 재주문
- 즐겨찾기 목록 → 주문서로 바로 전환

---

### Phase 3 — 플랫폼 확장

#### 4-8. 검색 고도화
- 한/영 동의어 매핑 ("까베르네" → Cabernet Sauvignon)
- PostgreSQL 전문 검색(Full-Text Search)
- 자동완성 드롭다운

#### 4-9. 마케팅 도구
- 관리자가 큐레이션 섹션 직접 편집
- 이달의 추천, 시즌 기획전, 신상품 배너

#### 4-10. 공급자 성과 리포트
- 상품별 조회수, 찜수, 주문 전환율
- AI 소믈리에에서 추천된 횟수

## 5. 데이터 모델 변경 계획

### 추가 필요 테이블
```
orders          — 주문 헤더
order_items     — 주문 상품 라인
suppliers       — 공급자 프로필
favorites       — 수요자 즐겨찾기
wine_images     — 상품 이미지 (멀티)
```

### 기존 테이블 변경
```
wines
  + image_url       (대표 이미지)
  + status          (판매중 / 품절 / 단종)
  + supplier_id     (공급자 연결)
  + stock_updated_at (재고 업데이트 시각)

users
  + user_type       (buyer / supplier / admin)
  + company_name    (상호명)
  + phone           (연락처)
  + approved        (공급자 승인 여부)
```

## 7. joomoonAI 앱 방향 (메모)

> 별도 PRD로 분리 예정. 현재는 방향만 기록.

- **성격**: 수요자(소매상·호레카) 중심 모바일 앱
- **핵심 기능 후보**
  - AI 소믈리에와 대화로 즉시 주문
  - 바코드/라벨 스캔으로 상품 검색
  - 단골 공급자 빠른 재주문
  - 주문 현황 실시간 알림 (푸시)
- **기술 방향**: React Native 또는 웹뷰 기반 (현재 웹 코드 재활용 가능성 검토 필요)
- **우선순위**: 웹 플랫폼 안정화 후 착수
