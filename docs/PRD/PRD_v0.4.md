## 1. 제품 비전

### 브랜드
**joomoon.ai** *(joomoon = 주문, 注文)*
- 도메인 후보: `joomoon.ai` (검토 중, 변경 가능)
- 한 줄 정의: *"AI가 연결하는 B2B 주류 거래 플랫폼"*

### 핵심 가치
1. **발견(Discovery)**: AI 소믈리에가 적합한 상품을 능동적으로 제안
2. **연결(Connection)**: 공급자와 수요자를 직접 연결하는 마켓플레이스
3. **신뢰(Trust)**: 정확한 상품 데이터와 투명한 거래 구조

## 2. 타겟 사용자

### 2-1. 수요자 — Buyer (B2B)
**누구**: 와인·전통주 소매상, 호텔·레스토랑·카페(호레카)

**핵심 니즈**
- 상황(시즌, 행사, 손님 유형)에 맞는 상품을 빠르게 찾고 싶다
- 소비자에게 상품을 설명할 수 있는 **스토리와 영업 멘트**가 필요하다
- 소량/다품종 주문이 가능하고 납기가 예측 가능해야 한다

### 2-2. 공급자 — Supplier (B2B)
**누구**: 와인 도매상, 유통업체, 수입상 (중소규모)

**핵심 니즈**
- 기본 정보만 올리면 **마케팅 자료가 자동 생성**되길 원한다
- 신규 바이어를 디지털로 발굴하고 싶다
- 재고 상황에 따라 추천 노출을 조절하고 싶다

### 2-3. 내부 관리자 — Admin
**누구**: 플랫폼 운영팀, 마케팅팀

**핵심 니즈**
- 사용자 로그 분석으로 AI 소믈리에를 지속 개선
- 상품 데이터 품질 관리
- KeywordLib 보강 및 임베딩 재생성

## 3. 확정된 결정사항

### 3-1. 수익 모델
- 거래액 기준 **% 수수료** + 공급자 **월 구독료** 혼합
- 구체적 비율은 크루 협의 후 확정 예정

### 3-2. 결제 방향 — 하이브리드 단계별 전환

```
Phase 1~2          Phase 2~3            Phase 3~
━━━━━━━━━━━━━      ━━━━━━━━━━━━━        ━━━━━━━━━━━━━━
주문 요청 방식       청구서 자동 생성      PG 연동
(오프라인 결제)      (거래 데이터 축적)    (수수료 자동화)
```

> ⚠️ **법무 검토 선행 필요**: 주류 B2B 온라인 중개 허용 범위 확인 후 방식 확정
> 상세 분석: [결제_케이스_분석.md](../biz_research/결제_케이스_분석.md)

### 3-3. 페르소나 시스템 — 보류
- 실험 결과 변별력 확보 실패
- 추후 재연구 예정 (Phase 3 이후)
- 현재는 단일 소믈리에 말투로 진행

### 3-4. 배달 연결 기능 — 제거
- 더데어(무인 와인 공간) 사업 연계 목적이었으나 현재 사업에 불필요
- **음식 페어링 자체는 중요한 팩터로 유지**

### 3-5. 데이터 전략
- 현재 데이터로 개발 진행 + 가짜 데이터 500개로 확대
- `is_sample = true` 플래그로 구분 관리
- 공급자 입점 후 진짜 데이터로 단계적 교체

### 3-6. AI 검색 — 하이브리드 방식
- GPT-4o-mini (의도 분석) + GPT-4o (스토리텔링)
- pgvector는 와인 1,000개 초과 시 도입 (리스크 없음)

### 3-7. 공급자 온보딩
- 직접 가입 신청 → 관리자 승인
- 초기에는 운영팀이 대신 등록 병행

### 3-8. joomoonAI 앱
- 웹 Phase 1 완료 후 별도 PRD 작성 후 착수

## 4. 핵심 기능 정의

### 4-1. AI 소믈리에

#### 기본 원칙 (블루프린트 v1.2.2 기반)

**추천은 항상 Top3 먼저**
```
조건에 맞는 와인 N개 중 → 3개를 먼저 추천
"총 12개가 있어요. 이 중 3가지를 먼저 추천드릴게요!"
→ 사용자 요청 시 전체 리스트 확장 가능
```

**단계적 추천 흐름**
```
① 자연어 또는 버튼 검색
        ↓
② AI가 Occasion 추출 및 구조화
  사람용: "데이트하기 좋은 로맨틱한 와인"
  머신용: { occasion: "date", mood: "romantic", party_size: 2 }
        ↓
③ DB에서 후보 와인 검색
  (강화 데이터 태그 + 조건 필터)
        ↓
④ Top3 교정 추천 카드 제시
        ↓
⑤ 와인 선택 시 → 상세 정보 + 페어링 추천
        ↓
⑥ 장바구니(주문 요청서) 담기
```

**진입 방식 2가지 (항상 전환 가능)**
```
채팅 모드: 자연어 대화로 추천
버튼 검색 모드: 타입 / 국가 / 품종 / 가격 필터 선택
```

#### 음식 페어링
- 와인 선택 후 어울리는 음식 추천
- 구체적인 음식명 + 이유 포함
- (배달 연결 기능은 제거)

---

### 4-2. 강화 데이터 파이프라인 ⭐ 핵심

**개념**: 공급자가 기본 정보만 입력하면 마케팅 자료가 자동 생성됨

```
공급자 입력 (최소한)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
와인명 / 타입 / 국가 / 가격
산미·바디·당도·타닌 (1~5)
생산자 이름 (선택)
        ↓
AI Agent 자동 생성
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
① 테이스팅 노트 (감성적, 문학적)
② AI 소믈리에 추천 멘트
③ 영업 멘트 (소매상 → 소비자에게 설명할 말)
④ 음식 페어링 (상세)
⑤ 상황별 태그 (mood_tags, scene_tags)
⑥ 초보자용 / 전문가용 설명 (2버전)
⑦ SNS 짧은 카피 (선택)
        ↓
단계별 운영
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Phase 1: 휴먼루프
  생성 → 관리자/공급사 검토·수정 → 승인 → 저장

Phase 2: 반자동
  생성 → 품질 점수 높으면 자동 승인 → 낮으면 검토 요청

Phase 3: 에이전트
  생성 → 자동 저장 → 사후 모니터링만
```

**비용**: 와인 1개당 약 ₩2 (GPT-4o-mini), 1회성

### 4-3. 주문 요청 시스템

**주문요청서 = 결제 아님 (블루프린트 원칙)**
```
장바구니 = 저장 + 주문요청서 생성 목적
수량 / 사입 희망가 / 납기 / 요청 메모 포함
```

**주문 흐름**
```
바이어: 와인 선택 → 장바구니 → 주문요청서 작성
        ↓
공급사: 주문 수신 → 수락 / 조건 협의 / 거절
        ↓
플랫폼: 상태 추적 (접수→수락→처리중→완료)
        ↓
결제: 오프라인 (계좌이체 + 세금계산서)
```

---

### 4-4. 공급자 포털

```
가입 신청 → 관리자 승인
        ↓
상품 등록 (기본 정보 입력)
        ↓
강화 데이터 자동 생성 (파이프라인)
        ↓
검토·수정 (휴먼루프)
        ↓
승인 → 플랫폼에 노출
        ↓
주문 수신 → 처리 → 상태 업데이트
```

**재고 관리 원칙**
```
판매중: AI 소믈리에 추천 포함
품절: AI 추천 제외 (DB에는 보존)
단종: AI 추천 제외 (DB에는 보존 — 데이터 자산)
```

### 4-5. 운영 루프 (관리자)

블루프린트에서 가져온 핵심 운영 개념

```

  어떤 질문에 추천이 잘 됐나? 못 됐나?
        ↓
Occasion Lexicon 보강
  새로운 표현 패턴 추가
        ↓
KeywordLib 업데이트
        ↓
임베딩 재생성 (pgvector 도입 후)
        ↓
반복
```

**관리자 어드민 필요 기능**
- 사용자 대화 로그 조회
- KeywordLib CSV 업로드
- 와인 데이터 CSV 업로드
- 강화 데이터 일괄 생성 버튼
- 임베딩 재생성 버튼 (Phase 3)

## 6. AI 소믈리에 기술 아키텍처

```
사용자 질문
    ↓

  사람용 태그: "로맨틱한 데이트용"

  → 관련 와인 20개 추출
    ↓
③ GPT-4o — Top3 선정 + 스토리텔링 (₩31/건)
  emotion_story, sales_pitch 활용
  → 문학적·감성적 추천 생성
    ↓
총 비용: 메시지당 약 ₩36
```

### pgvector 전환 기준

```
와인 1,000개 초과 시 ② 단계만 변경:
DB 태그 검색 → pgvector 의미 유사도 검색
(① ③ 동일, 비용 동일, 코드 변경 최소)
```
