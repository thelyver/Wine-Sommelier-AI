## 1. 현재 joomoon.ai 추천 로직 (코드 기반 분석)

### 실제 흐름

```
사용자: "비 오는 날 혼자 위로받고 싶어"
         │
         ▼
① analyzeQueryForFilters()   ← 순수 코드 (GPT 없음)
   DB의 키워드 테이블과 문자열 비교
   - tasteLevels: "달콤한", "드라이" 등 매칭
   - priceRanges: "저렴한", "고급" 등 매칭
   - keywordLib:  "레드", "화이트", "프랑스" 등 매칭
   - occasions:   "데이트", "파티" 등 매칭
   
   결과: {} (빈 객체 — "비 오는 날", "위로" 는 매칭 안 됨!)
         │
         ▼
② DB 쿼리
   filters가 없으면 → getWinesForContext(50)
   그냥 상위 50개 와인을 순서대로 꺼냄
         │
         ▼
③ 와인 컨텍스트 문자열 생성
   "- 샤또 마고 (ta001): RED, France, Cabernet Sauvignon,
    150,000원, 단맛:1/5 산미:3/5 바디:5/5 탄닌:4/5, 우아하고..."
         │
         ▼
④ gpt-4o 호출
   시스템 프롬프트 + 50개 와인 스펙 + 사용자 질문
   → GPT가 스펙만 보고 감성 이야기를 "만들어냄"
         │
         ▼
⑤ 응답 스트리밍
```

### 현재 구조의 핵심 약점

| 단계 | 문제 |
|------|------|
| ① 필터 추출 | 감성어 ("위로", "설레는", "비 오는 날") 매칭 불가 |
| ② DB 검색 | 감성/상황 기반 검색 불가, 그냥 50개 순서대로 |
| ③ 컨텍스트 | 와인 스펙만 있음. 감성 소재가 없음 |
| ④ GPT 역할 | 스펙→감성 번역을 GPT가 혼자 다 해야 함 (부담↑, 일관성↓) |

## 2. 세 가지 아키텍처 비교

### Type A — GPT Only

```
사용자 질문
    │
    ▼
GPT (와인 지식 내장)
    │
    ▼
추천 답변
```

**실제 사례:** ChatGPT에 "비 오는 날 와인 추천해줘" 라고 물어보는 것

| 항목 | 평가 |
|------|------|
| 감성 이해 | ✅ 매우 뛰어남 |
| 추천 다양성 | ✅ 풍부한 지식 |
| **재고 연동** | ❌ 불가능 (없는 와인 추천) |
| **가격 정확도** | ❌ 환각(hallucination) |
| **반복 일관성** | ❌ 같은 질문에 다른 답변 |
| 비용 | 중간 |
| **B2B 플랫폼 적합성** | ❌ 실제 거래 불가 |

**결론:** 정보 서비스는 가능하지만, 실제 구매 연결 불가

---

### Type B — GPT + RDB 키워드 검색 (현재 joomoon.ai)

```
사용자 질문
    │
    ▼
키워드 매칭 (코드)
    │
    ▼
SQL 필터 검색
    │
    ▼
GPT (스펙 데이터 기반 응답)
    │
    ▼
추천 답변
```

**실제 사례:** 대부분의 AI 커머스 추천 시스템

| 항목 | 평가 |
|------|------|
| 감성 이해 | 🟡 제한적 (등록된 키워드만) |
| 추천 다양성 | 🟡 필터 결과 내에서만 |
| 재고 연동 | ✅ 실제 DB 와인만 추천 |
| 가격 정확도 | ✅ DB 기반 정확 |
| 반복 일관성 | 🟡 필터는 일관, 표현은 랜덤 |
| B2B 플랫폼 적합성 | 🟡 가능하지만 경험 평범 |

**결론:** 재고 연동은 되지만, "비 오는 날" 같은 감성 질문에 약함

---

### Type C — GPT + RDB + 강화데이터 (목표 joomoon.ai)

```
사용자 질문
    │
    ▼
① GPT-4o-mini: 의도/감성 구조화
   "비 오는 날 위로" →
   { mood: ["위로", "차분"], scene: ["혼술"], type: null }
    │
    ▼
② DB 검색: mood_tags / scene_tags 기반
   "혼술" 태그 보유 와인 20개 추출
    │
    ▼
③ 강화 컨텍스트 구성
   스펙 + emotion_story + sales_pitch + pairing_detail
    │
    ▼
④ GPT-4o: 스토리텔링 생성
   (소재가 이미 있으므로 일관되고 풍부한 이야기 가능)
    │
    ▼
추천 답변
```

| 항목 | 평가 |
|------|------|
| 감성 이해 | ✅✅ 최고 (GPT가 의도 해석) |
| 추천 다양성 | ✅ mood/scene 태그 기반 |
| 재고 연동 | ✅ 실제 DB 와인만 추천 |
| 가격 정확도 | ✅ DB 기반 정확 |
| 반복 일관성 | ✅ 강화 데이터가 앵커 역할 |
| B2B 플랫폼 적합성 | ✅✅ 소매상 영업 멘트까지 제공 |

**결론:** 세 가지 중 유일하게 "감성 → 실제 구매" 연결 가능

## 3. 강화 데이터가 만드는 차이 (핵심)

### 같은 질문, 다른 소재

**질문:** "비 오는 날 혼자 마시기 좋은 와인은?"

**Type B (스펙만 있을 때) GPT가 받는 소재:**
```
- 샤또 피숑 라랑드 2018 (RED, France, 120,000원)
  단맛:1 산미:3 바디:5 탄닌:4
  "보르도의 클래식한 레드 와인"
```

**Type C (강화 데이터 있을 때) GPT가 받는 소재:**
```
- 샤또 피숑 라랑드 2018 (RED, France, 120,000원)
  단맛:1 산미:3 바디:5 탄닌:4
  
  emotion_story: "빗소리를 배경음악 삼아 혼자 앉은 저녁,
                  첫 잔을 따르면 블랙베리와 삼나무 향이
                  조용히 방 안을 채웁니다."
  
  mood_tags: ["위로", "차분함", "성찰"]
  scene_tags: ["혼술", "실내", "저녁"]
  
  sales_pitch: "손님께 이렇게 말씀드리세요:
                '혼자만의 시간을 더 특별하게 만들어줄
                와인입니다. 한 병 열면 저절로 느려지게 됩니다.'"
```

GPT는 소재를 **발명**하는 게 아니라 **편집**하면 됩니다.
→ 품질 일관성 대폭 향상

## 4. 강화 데이터 생성 계획

### 생성할 항목 (7가지)

```typescript
mood_tags: string[]      // ["위로", "차분함", "설렘"] — 감성 태그
scene_tags: string[]     // ["혼술", "데이트", "비즈니스 디너"] — 상황 태그
emotion_story: text      // 2~3문장 감성 상황 묘사
sales_pitch: text        // 소매상이 소비자에게 설명할 멘트
pairing_detail: text     // 구체적 음식 페어링
occasion_json: jsonb     // { primary: "혼술", secondary: ["실내", "저녁"] }
keyword_cloud: string[]  // 검색 강화용 연관 단어들
```

### 생성 시점

| 시점 | 방법 | 비용 |
|------|------|------|
| 기존 231개 일괄 처리 | 1회 배치 스크립트 | 약 ₩50 |
| 신규 와인 등록 시 | 자동 생성 (공급자/관리자) | 와인당 ₩2 |
| 정보 수정 시 | 재생성 선택 가능 | 와인당 ₩2 |

### 생성 프롬프트 (GPT-4o-mini 사용)

```
[시스템] 당신은 와인 마케팅 전문가입니다.
아래 와인 정보를 보고 감성 마케팅 데이터를 JSON으로 생성하세요.

[입력]
이름: {nameKr}
타입: {type} / 국가: {nation} / 품종: {varieties}
단맛:{sweet}/산미:{acidity}/바디:{body}/탄닌:{tannin} (1-5)
가격: {price}원
기존 설명: {description}

[출력 JSON]
{
  "mood_tags": ["감성1", "감성2", "감성3"],
  "scene_tags": ["상황1", "상황2", "상황3"],
  "emotion_story": "2~3문장의 감성 상황 묘사",
  "sales_pitch": "소매상이 소비자에게 설명할 멘트 1~2문장",
  "pairing_detail": "구체적인 음식 페어링",
  "keyword_cloud": ["연관어1", "연관어2", ...]
}
```
