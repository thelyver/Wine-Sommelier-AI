## 배경

- B2B 주류 유통 플랫폼 특성상 결제 방식이 일반 커머스와 다름
- 주류는 면허가 필요한 **특수 품목** → 법적 검토 필수
- B2B 관행: 후불 세금계산서, 월말 정산, 신용거래 多
- 플랫폼 내 결제는 반드시 이뤄져야 함 (외부 결제만으로는 부족)

## Case 1 — 주문 요청 + 오프라인 결제

> **현재 MVP로 확정된 방식**

### 흐름

```
바이어
  플랫폼에서 주문 요청서 작성
  (수량 / 사입 희망가 / 납기 / 요청 메모)
        ↓
공급사
  주문 확인 → 수락 또는 조건 협의
        ↓
결제
  플랫폼 밖에서 진행
  (계좌이체 + 세금계산서 발행)
        ↓
플랫폼
  주문 상태만 기록 (접수 → 수락 → 완료)
```

### 평가

| 항목 | 내용 |
|------|------|
| 장점 | 가장 빠른 구현 / B2B 업계 익숙한 방식 |
| 단점 | 플랫폼이 결제 흐름 파악 불가 / 수수료 수취 어려움 |
| 개발 비용 | 최소 (PG 없음) |
| 법적 리스크 | 낮음 |
| **적합 시점** | **지금 바로 (Phase 1~2)** |

## Case 2 — PG 연동 (카드·계좌이체)

### 흐름

```
바이어
  플랫폼 내에서 카드 또는 계좌이체로 결제
        ↓
플랫폼
  결제 수취 → 정산 주기에 공급사 송금
  수수료 차감 후 정산
        ↓
공급사
  정산금 수령 + 세금계산서 발행
```

### 평가

| 항목 | 내용 |
|------|------|
| 장점 | 완전한 거래 추적 / 수수료 자동 수취 / 데이터 축적 |
| 단점 | PG사 심사 필요 (1~2개월) / 정산 시스템 개발 복잡 |
| 수수료 | 결제액의 약 1.98~2.5% (토스페이먼츠 기준) |
| 개발 비용 | 높음 |
| **적합 시점** | **Phase 3 (거래량 안정 후)** |
| 추천 PG사 | 토스페이먼츠 (B2B 정산 기능 보유) |

## Case 3 — 청구서(인보이스) 방식

> **B2B에서 가장 자연스러운 중간 단계**

### 흐름

```
바이어
  플랫폼에서 주문 확정
        ↓
플랫폼
  자동으로 견적서 / 청구서 생성 및 발송
        ↓
바이어
  청구서 확인 후 계좌이체
        ↓
공급사
  세금계산서 발행
        ↓
플랫폼
  입금 확인 후 주문 상태 업데이트
```

### 평가

| 항목 | 내용 |
|------|------|
| 장점 | B2B에서 가장 자연스러운 방식 / 신용거래 가능 / 구현 단순 |
| 단점 | 입금 확인 수동 처리 필요 / 미수금 리스크 |
| 개발 비용 | 낮음~중간 |
| **적합 시점** | **Phase 2 (Case 1 이후 단계)** |

## Case 4 — 에스크로 방식

### 흐름

```
바이어
  플랫폼에 결제 (선입금)
        ↓
플랫폼
  결제금 보관 (에스크로)
        ↓
배송 완료 확인 (바이어 확정)
        ↓
플랫폼
  공급사에 정산 (수수료 차감)
```

### 평가

| 항목 | 내용 |
|------|------|
| 장점 | 바이어/공급사 모두 신뢰 보호 / 분쟁 해결 구조 명확 |
| 단점 | 가장 복잡한 구현 / 에스크로 라이선스 필요할 수 있음 |
| 개발 비용 | 매우 높음 |
| **적합 시점** | 플랫폼 신뢰도 구축 후 (장기) |

## 주류 B2B 특수 고려사항

### ① 법적 이슈 (가장 중요)

```
주류 온라인 거래는 주세법 및 식품위생법 규제 대상

확인 필요 사항:
- 플랫폼이 "중개자"인지 "판매자"인지 법적 정의
- 온라인 주류 판매 허용 범위 (B2B는 B2C와 다름)
- 세금계산서 발행 의무 주체

→ 법무 검토를 결제 방식 결정 전에 먼저 진행 권장
```

### ② B2B 신용거래 관행

```
주류 유통 업계 실제 관행:
- 월말 일괄 정산 (익월 10~15일 입금)
- 어음 또는 외상 거래
- 신규 거래처는 선불, 기존 거래처는 후불

→ 즉시 결제 요구 시 공급사·바이어 모두 거부감
→ 신용 한도 설정 기능 필요할 수 있음
```

### ③ 최소 주문 수량 (MOQ)

```
공급사마다 MOQ 상이:
- 와인: 보통 6병(1box) 또는 12병(1case) 단위
- 주문 요청서에 "조건 협의" 기능 필요
```

### ④ 부가세 처리

```
주류 B2B 거래:
- 공급가 + 부가세 10% 별도 표기 필요
- 세금계산서 발행 의무
- 플랫폼 수수료에도 부가세 별도 청구
```
