### [Task 1] 소피 답변 품질 강화 — 프롬프트 v2 🔴 최우선

**목표**: "잘 어울립니다" 같은 이유 없는 결론형 표현 완전 제거

현재 남아있는 문제 표현들:
```
❌ "스테이크와 잘 어우러져 맛을 더욱 풍부하게 해줍니다"
❌ "특별한 순간을 함께 나누고 싶게 만듭니다"
❌ "부드럽게 퍼지는 탄닌" (탄닌이 어떻게 느껴지는지 없음)
```

개선 방향:
- 금지 표현 목록 확대 (더 구체적으로)
- few-shot 좋은 예시 3개 → 6개로 늘리기
- 강화 데이터(emotionStory, salesPitch) 활용 지시 강화
- "~감이 있습니다" "~를 느낄 수 있습니다" 등 두루뭉술 표현 금지 추가

---

### [Task 2] 와인 상세 모달 — 강화 데이터 연결 🟡 중요

**목표**: 상세 모달의 어색한 원본 CSV 텍스트 → 소피가 만든 감성 데이터로 교체

| 탭 | 현재 (원본 CSV) | 변경 후 (강화 데이터) |
|----|----------------|---------------------|
| 개요 | summary (기계번역 느낌) | emotionStory (감성 묘사) |
| 테이스팅 | tastingNote (어색) | salesPitch (소피 영업 멘트) |
| 페어링 | pairing (단순 목록) | pairingDetail (구체적 설명) |

변경 파일: `client/src/components/wine-detail-modal.tsx`

## 소피(Sofy) AI 로드맵

> 소피의 품질은 단계적으로 개선됩니다

```
현재 (Type C 초기)
  · GPT-4o + 강화 데이터(emotionStory/salesPitch) 연결
  · 3단계 공식 내부 적용, 자연 단락 출력

  · 프롬프트 v2: 모호 표현 완전 제거

↓ Sprint 2 목표

  · 소피가 다양한 페어링 시나리오를 구체적으로 설명

↓ Phase 3 이후 (pgvector)
  · 1,000종+ 시 의미 기반 검색
  · "비슷한 느낌의 와인 더 찾아줘" 기능

```

